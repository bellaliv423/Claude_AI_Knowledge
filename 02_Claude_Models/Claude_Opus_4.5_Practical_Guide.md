## Opus 4.5 소개 / 介紹

### 🇰🇷 한국어

Claude Opus 4.5는 현재까지 **가장 지능적인 Claude 모델**입니다. 복잡하고 다단계 작업에서 차이가 나타납니다.

**핵심 특징:**
- 요청을 깊이 이해하고 파일을 심층 분석
- 긴 작업 전체에서 컨텍스트 유지
- 어려운 문제를 논리적으로 추론
- 그것을 반영한 결과물 제공

### 🇹🇼 繁體中文

Claude Opus 4.5 是迄今為止**最智能的 Claude 模型**。在複雜的多步驟工作中差異尤為明顯。

**核心特點：**
- 深入理解您的請求並深度分析檔案
- 在長時間工作中保持上下文
- 對困難問題進行邏輯推理
- 提供反映這些特點的結果

## 핵심 개선 사항 / 核心改進

### 🇰🇷 한국어

| 개선 영역 | 설명 |
|----------|------|
| **끊기지 않는 대화** | 컨텍스트 제한에 도달하거나 처음부터 다시 시작할 필요 없이 확장된 프로젝트 진행 가능 |
| **바로 사용 가능한 출력물** | 작동하는 수식이 있는 스프레드시트, 구조와 시각적 요소가 있는 프레젠테이션, 세심한 디자인의 프론트엔드 생성 |
| **궤도를 유지하는 코딩** | Claude가 컨텍스트를 잃지 않고 확장된 코딩 작업 처리. 어려운 문제 해결에 끈기 있음 |
| **더 날카로운 시각** | 밀도 높은 다이어그램, 이미지, 비디오를 더 정확하게 처리 |

### 🇹🇼 繁體中文

| 改進領域 | 說明 |
|----------|------|
| **不中斷的對話** | 無需達到上下文限制或重新開始，可進行延伸專案 |
| **即用型輸出** | 創建具有有效公式的試算表、具有結構和視覺元素的簡報、精心設計的前端 |
| **保持軌道的編程** | Claude 不會失去上下文，處理延伸的編程任務。解決困難問題時很有毅力 |
| **更敏銳的視覺** | 更準確地處理密集的圖表、圖像和視頻 |

## Opus 4.5가 빛나는 영역 / 擅長領域

- **여러 결과물을 개발하는 프로젝트**

- **跨越多個工具和來源的研究**
- **經過多輪修訂而演進的輸出**
- **一起上傳和分析多個檔案**
- **開發多個交付物的專案**
- **早期選擇影響後續實施的決策**

## 스프레드시트 활용 / 試算表應用

### 🇰🇷 한국어

#### 고급 기능 요청
Opus 4.5는 다음을 포함한 스프레드시트 구축 가능:
- 피벗 테이블
- 임베디드 차트
- 스파크라인
- 교차 탭 참조
- 조건부 서식

#### 처음부터 또는 기존 것 확장
- 리서치를 수행하고 스프레드시트 형태로 결과 제공
- 기존 시트에 구조를 유지하면서 추가

#### 직접 사용 가능한 모델 구축
- 입력이 변경되면 업데이트되는 수식 요청 (정적 값이 아님)
- 명명된 범위와 주석으로 나중에 감사 및 수정 가능

#### 형식 간 변환
- CSV를 서식이 적용된 스프레드시트로
- PDF나 문서에서 테이블 추출
- 한 레이아웃에서 다른 레이아웃으로 데이터 재구성

### 🇹🇼 繁體中文

#### 請求進階功能
Opus 4.5 可以建立包含以下內容的試算表：
- 樞紐分析表
- 嵌入式圖表
- 走勢圖
- 跨表引用
- 條件格式

#### 從頭開始或擴展現有的
- 進行研究並以試算表形式提供結果
- 在保留結構的同時添加到現有工作表

#### 建立可直接使用的模型
- 請求輸入變更時會更新的公式（非靜態值）
- 使用命名範圍和註解以便日後審計和修改

#### 格式間轉換
- 將 CSV 轉換為格式化試算表
- 從 PDF 或文件中提取表格
- 將數據從一種佈局重組為另一種

## 프레젠테이션 활용 / 簡報應用

### 🇰🇷 한국어

#### 시각적 스토리 전달
- 임베디드 차트, 이미지, 콜아웃 박스, 다이어그램 추가
- 각 슬라이드가 전체 아크에서 무엇을 달성해야 하는지 설명 (단순히 어떤 콘텐츠가 아니라)

#### 스타일 맞추기
- 브랜드 색상, 폰트, 레이아웃 선호도를 미리 지정
- 기존 덱을 업로드하고 같은 스타일로 슬라이드 추가 요청
- 마음에 드는 단일 슬라이드를 공유하고 전체적으로 매칭 요청

#### 거친 자료에서 구축
- 회의 노트, 아웃라인, 문서를 구조화된 프레젠테이션으로 변환
- 핵심 포인트 추출 후 지원 시각 자료와 함께 내러티브로 구성

#### 내보내기 전 확인
- 겹치는 것이 없는지, 텍스트가 슬라이드를 벗어나지 않는지, 서식이 일관되는지 확인 요청

### 🇹🇼 繁體中文

#### 講述視覺故事
- 添加嵌入式圖表、圖像、標註框、圖表
- 描述每張投影片在整體敘事中應該達成什麼（不僅僅是包含什麼內容）

#### 匹配您的風格
- 預先指定品牌顏色、字體和版面偏好
- 上傳現有簡報並要求以相同風格添加投影片
- 分享您喜歡的單張投影片，要求全程匹配

#### 從粗糙材料建立
- 將會議記錄、大綱或文件轉換為結構化簡報
- 提取關鍵點後與支援視覺元素一起組織成敘事

#### 導出前檢查
- 要求確認沒有重疊、文字不會超出投影片、格式保持一致

## 문서 작업 활용 / 文件工作應用

### 🇰🇷 한국어

#### Claude를 편집자로 활용
- 대신 쓰는 것이 아니라 초안을 검토하고 **추적 변경 및 주석**과 함께 문서에 직접 제안

#### 편집 전 분석
- Opus에게 의도한 청중으로서 초안을 읽도록 요청:
  - 무엇이 혼란스러울까?
  - 무엇이 빠져있을까?
  - 무엇에 반박할 수 있을까?
- 그런 다음 그 간극을 메우도록 수정 요청

#### 예시에서 톤 맞추기
- 원하는 목소리가 담긴 단락이나 문서 업로드
- **스타일을 설명하는 것보다 보여주는 것이 더 효과적**

### 🇹🇼 繁體中文

#### 將 Claude 作為編輯使用
- 不是代替您寫作，而是審閱您的草稿並在文件中直接提出**追蹤變更和註解**建議

#### 編輯前分析
- 要求 Opus 以您的目標受眾身份閱讀草稿：
  - 什麼會讓他們困惑？
  - 缺少什麼？
  - 他們可能會反駁什麼？
- 然後要求修改以解決這些差距

#### 從範例匹配語調
- 上傳包含您想要的風格的段落或文件
- **展示風格比描述風格更有效**

## 코드 및 디자인 / 代碼與設計

### 🇰🇷 한국어

Opus 4.5는 **코딩 분야의 프론티어 모델**입니다.

#### 특징
- 더 길고 복잡한 작업 처리
- 작동하는 결과에 도달하기 위해 더 적은 조정 필요
- 문제를 통해 방향 유지
- 접근 방식이 실패하면 조정

#### 세련된 디자인 얻기
- 시각적 컴포넌트가 더 고려되고 잘 만들어짐
- 원하는 미학 설명: 색상 팔레트, 간격, 계층 구조, 전반적인 느낌

#### 참조 맞추기
- 마음에 드는 스크린샷을 공유하고 매칭 요청
- 원하는 스타일을 설명하는 것보다 쉬운 방법

#### 세부 사항 반복
- 첫 구현 후 피드백으로 개선:
  - 간격 조이기
  - 색상 부드럽게
  - 요소 나타날 때 애니메이션 추가
- 느슨한 지시에서도 의도를 파악하고 조정

### 🇹🇼 繁體中文

Opus 4.5 是**編程領域的前沿模型**。

#### 特點
- 處理更長、更複雜的任務
- 達到有效結果需要更少的引導
- 在問題中保持方向
- 方法失敗時進行調整

#### 獲得精緻設計
- 視覺組件更加考究和精心製作
- 描述您想要的美學：調色板、間距、層次結構、整體感覺

#### 匹配參考
- 分享您喜歡的截圖並要求匹配
- 比描述您想要的風格更簡單的方法

#### 迭代細節
- 第一次實現後用反饋改進：
  - 收緊間距
  - 軟化顏色
  - 元素出現時添加動畫
- 即使是鬆散的指示也能理解意圖並相應調整

## 모델 선택 가이드 / 模型選擇指南

### 🇰🇷 한국어

> **Opus 4.5는 속도보다 깊이와 품질을 우선시합니다.** 개별 응답은 더 오래 걸리지만, 추론 방식이 더 효율적이어서 답에 더 직접적으로 도달합니다.

#### 모델 비교

| 모델 | 용도 | 특징 |
|------|------|------|
| **Opus 4.5** | 복잡한 추론, 품질이 중요한 작업 | 가장 지능적, 깊이 우선 |
| **Sonnet 4.5** | 일상적 대화, 루틴 작업 | 빠르고 대부분 작업에 유능 |
| **Haiku 4.5** | 빠른 질문, 간단한 작업 | 가장 빠름 |

#### Opus 4.5로 시작해야 할 때

- [ ] 작업이 서로 쌓이는 **여러 단계**를 가짐
- [ ] **세련된 결과물** 필요 (문서, 프레젠테이션, 작동하는 수식이 있는 스프레드시트)
- [ ] 이미지, 차트, 다이어그램, 시각 정보의 **심층 분석** 포함
- [ ] 순서와 논리가 중요한 문제에 대한 **신중한 추론** 필요
- [ ] 많은 메시지나 세션에 걸친 **프로젝트** 진행

#### Sonnet 4.5로 시작해야 할 때

- [ ] 간단한 질문에 대한 **빠른 답변** 필요
- [ ] **속도**가 깊이보다 중요
- [ ] 작업이 **루틴** (편집, 요약, 간단한 코드)
- [ ] **캐주얼한 대화**나 브레인스토밍
- [ ] 여러 단계에 걸친 확장된 추론이 **필요 없음**

### 🇹🇼 繁體中文

> **Opus 4.5 優先考慮深度和品質而非速度。** 單個回應需要更長時間，但推理方式更高效，更直接地得出答案。

#### 模型比較

| 模型 | 用途 | 特點 |
|------|------|------|
| **Opus 4.5** | 複雜推理、品質重要的工作 | 最智能、深度優先 |
| **Sonnet 4.5** | 日常對話、例行任務 | 快速且大多數工作都能勝任 |
| **Haiku 4.5** | 快速問題、簡單任務 | 最快 |

#### 何時從 Opus 4.5 開始

- [ ] 任務有相互建立的**多個階段**
- [ ] 需要**精緻的交付物**（文件、簡報、具有有效公式的試算表）
- [ ] 工作涉及圖像、圖表、圖表、視覺信息的**深入分析**
- [ ] 需要對順序和邏輯重要的問題進行**謹慎推理**
- [ ] 正在進行跨越多個訊息或會話的**專案**

#### 何時從 Sonnet 4.5 開始

- [ ] 需要對簡單問題的**快速答案**
- [ ] **速度**比深度更重要
- [ ] 任務是**例行的**（編輯、摘要、簡單代碼）
- [ ] **休閒對話**或腦力激盪
- [ ] 不需要跨多個步驟的**延伸推理**

## 핵심 인사이트 / 核心洞見

### 🇰🇷 한국어

> **"Opus 4.5에게 누구를 위한 것인지, 어떻게 사용할 것인지 말해주면 구조와 세부사항에 대해 더 나은 결정을 내릴 수 있습니다."**

- 스타일을 **설명하기보다 보여주기**
- 단순히 콘텐츠가 아닌 **목적** 설명
- 개별 응답 속도보다 **총 반복 비용** 고려
