## 개요 / 概述

### 🇰🇷 한국어

Boris Cherny는 Claude Code의 창시자이자 책임자입니다. Claude Code는 출시 6개월 만에 약 10억 달러의 연간 매출을 기록했습니다.

그가 Anthropic에서 일하는 이유:
> "AI가 사회에 미치는 변혁적 영향이 잘 진행되도록 아주 작은 방식으로나마 기여하고 싶었습니다"

### 🇹🇼 繁體中文

Boris Cherny 是 Claude Code 的創建者和負責人。Claude Code 在推出僅六個月後就創造了約 10 億美元的年收入。

他在 Anthropic 工作的原因：
> 「我想以最微小的方式確保 AI 對社會的變革性影響能順利進行」

## 핵심 전략 6가지 / 六大核心策略

### 1. 병렬 세션 관리 / 並行會話管理

#### 🇰🇷 한국어
- **10-15개**의 Claude Code 세션을 동시에 실행
- 터미널에서 5개, 웹사이트에서 5-10개 운영
- AI를 단일 순차 도구가 아닌 **컴퓨팅 리소스처럼** 분산 활용
- 인지 부하를 리소스처럼 분배

#### 🇹🇼 繁體中文
- 同時運行 **10-15 個** Claude Code 會話
- 終端機 5 個，網頁 5-10 個
- 像**運算資源**一樣分散使用 AI，而非單一順序工具
- 像分配資源一樣分配認知負載

---

### 2. 모델 선택 전략 / 模型選擇策略

#### 🇰🇷 한국어
- 모든 코딩에 **Opus 4.5 + Thinking** 사용
- Sonnet보다 느리지만 더 높은 품질과 신뢰성 제공

> 💡 **핵심 철학:** "빠르지만 틀린 답보다 느리지만 맞는 답이 더 빠르다"

- 토큰당 비용이 아닌 **총 반복 비용** 최적화

#### 🇹🇼 繁體中文
- 所有編程工作使用 **Opus 4.5 + Thinking**
- 比 Sonnet 慢但提供更高的品質和可靠性

> 💡 **核心哲學：** 「快但錯的答案比慢但對的答案更慢」

- 優化**總迭代成本**，而非每個 token 的費用

### 3. CLAUDE.md를 통한 기관 메모리 / 透過 CLAUDE.md 建立機構記憶

#### 🇰🇷 한국어
- Git에 체크인되는 **공유 문서**
- Claude가 한 **실수를 기록**하여 반복 방지
- 팀 전체가 **주 여러 차례** 업데이트
- 살아있는 시스템 문서로 관리

**활용 예시:**
```markdown
# CLAUDE.md

## 이 프로젝트에서 주의할 점
- API 호출 시 반드시 에러 핸들링 포함
- 날짜 형식은 항상 ISO 8601 사용
- 테스트 없이 코드 커밋 금지

## Claude가 이전에 한 실수
- 2025-01-15: 환경변수 없이 직접 API 키 하드코딩 시도 → .env 사용 필수
- 2025-01-18: 한국 시간대 미적용 → KST timezone 명시 필요
```

#### 🇹🇼 繁體中文
- 存入 Git 的**共享文件**
- 記錄 Claude 的**錯誤**以防止重複
- 團隊**每週多次**更新
- 作為活文件管理

**使用範例：**
```markdown
# CLAUDE.md

## 此專案注意事項
- API 調用時務必包含錯誤處理
- 日期格式始終使用 ISO 8601
- 禁止未經測試就提交代碼

## Claude 之前犯過的錯誤
- 2025-01-15: 嘗試直接硬編碼 API 金鑰 → 必須使用 .env
- 2025-01-18: 未應用台灣時區 → 需明確指定 CST timezone
```

### 4. 코드 리뷰 통합 / 代碼審查整合

#### 🇰🇷 한국어
- PR(Pull Request) 리뷰 시 **Claude를 태그**
- 학습 내용을 CLAUDE.md에 추가
- 코드 리뷰를 **메타 작업**으로 취급
  - 단순히 변경사항 승인이 아닌
  - **개발 프로세스 자체를 개선**

#### 🇹🇼 繁體中文
- PR（Pull Request）審查時**標記 Claude**
- 將學習內容添加到 CLAUDE.md
- 將代碼審查視為**元工作**
  - 不僅僅是批准變更
  - 而是**改進開發流程本身**

### 5. 실행 전 계획 수립 (Plan Mode) / 執行前制定計劃

#### 🇰🇷 한국어
- Claude가 자동으로 편집하기 전에 **Plan Mode** 사용
- 명시적 의도와 제약 조건 설정
- 원치 않는 **연쇄 변경 방지**
- 구조화된 합의 후 진행

**사용 방법:**
```
/plan  # Plan Mode 진입
# 또는 설정에서 Plan Mode를 기본으로 설정
```

#### 🇹🇼 繁體中文
- 在 Claude 自動編輯前使用 **Plan Mode**
- 設定明確的意圖和限制條件
- 防止不必要的**連鎖變更**
- 結構化協議後再進行

**使用方法：**
```
/plan  # 進入 Plan Mode
# 或在設定中將 Plan Mode 設為預設
```

### 6. 전문화된 서브에이전트 활용 / 使用專門化的子代理

#### 🇰🇷 한국어
서브에이전트를 **모듈식 도구**처럼 활용:

| 용도 | 설명 |
|------|------|
| 사양 작성 | 요구사항 정의 전담 |
| 초안 작성 | 코드 초안 생성 |
| 단순화 | 복잡한 코드 리팩토링 |
| 검증 | 테스트 및 품질 확인 |

> 💡 **공식:** 전문화 + 제약 조건 = 신뢰성 향상

#### 🇹🇼 繁體中文
將子代理當作**模組化工具**使用：

| 用途 | 說明 |
|------|------|
| 規格撰寫 | 專責定義需求 |
| 草稿撰寫 | 生成代碼草稿 |
| 簡化 | 重構複雜代碼 |
| 驗證 | 測試和品質確認 |

> 💡 **公式：** 專門化 + 限制條件 = 提高可靠性

## 🌟 가장 중요한 원칙 / 最重要的原則

### 피드백 루프 제공 / 提供反饋迴路

#### 🇰🇷 한국어

> **Boris Cherny:** "아마도 Claude Code에서 훌륭한 결과를 얻기 위한 **가장 중요한 것**은 Claude에게 자신의 작업을 검증할 방법을 주는 것입니다. Claude가 그 피드백 루프를 가지면 **최종 결과 품질이 2-3배 향상**됩니다."

**피드백 루프 구현 방법:**
1. 테스트 실행 (`npm test`, `pytest` 등)
2. 빌드 검증 (`npm run build`)
3. 린트 체크 (`eslint`, `flake8` 등)
4. 타입 체크 (`tsc --noEmit`)

#### 🇹🇼 繁體中文

> **Boris Cherny:** 「獲得 Claude Code 優秀結果的**最重要的事**可能是給 Claude 一個驗證其工作的方法。如果 Claude 有那個反饋迴路，**最終結果的品質會提高 2-3 倍**。」

**實現反饋迴路的方法：**
1. 運行測試（`npm test`、`pytest` 等）
2. 建置驗證（`npm run build`）
3. Lint 檢查（`eslint`、`flake8` 等）
4. 類型檢查（`tsc --noEmit`）
